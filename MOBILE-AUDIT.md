## Critical findings

### 🔴 `/comparison` — VA-vs-Ramped table word-wraps mid-word

**Before:** With `width:30%` on the label column at ~390px viewport, cells like "Consistency", "Sick days & turnover", "Money-back guarantee", "Deep relationship building" wrap mid-word (e.g. "Consistenc / y", "Money- / back / guarantee"). Looks broken.

**Cause:** `.table-scroll` wrapper exists in markup (`comparison.html:202`) but `.cmp-table` had no `min-width`, so the table shrunk to fit the parent rather than overflowing.

**Fix shipped in this PR:**

- `cmp-table` gets `min-width:580px` on `<=640px` viewports — table stays full width and the wrapper becomes a horizontal scroll container.
- Added a `.table-scroll-hint` ("Swipe to see more →") shown only on mobile.
- Added a fade-out gradient on the right edge to signal the cutoff.
- Added `role="table"` and `aria-label` for screen readers.

## Medium findings

### ⚠️ `/resources` — double-encoded HTML entities in card titles

**Symptom:** Resource cards show literal text like `today&#8217;s edition of The Download` instead of `today's edition of The Download`.

**Cause:** Not a CSS/layout issue — the JS feed renderer is `escape`-ing content that's already entity-encoded at the source feed (Anthropic / OpenAI changelog feeds, etc.). Likely in the `fetch → render` pipeline for `/resources`.

**Fix path (separate PR — not in this commit):** In whatever JS renders the cards, run the title through a single decode (e.g., `new DOMParser().parseFromString(t, 'text/html').documentElement.textContent`) before injecting, or strip the over-aggressive escape.

### ⚠️ `/heroes` — internal page, variant nav cramped

**Status:** Internal hero variant comparison page, used during design. Not linked from main nav, not customer-facing. Low priority. Skip unless we share the URL externally.
