## Bugs found (with fixes staged locally)

### 1. CRITICAL — Booking calendar never shows time slots

**Symptom:** On `/book`, picking any future weekday shows *"No slots available — try another date."* Every date.

**Root cause:** `/api/availability` returns config metadata only:
```json
{"days_available":["Mon","Tue","Wed","Thu","Fri","Sat"],"start_hour":8,"end_hour":23,"slot_duration_min":30,"blocked_dates":[],"timezone":"America/Chicago"}
```
…but `book.html`'s `fetchSlots` was reading `d.slots` (which doesn't exist), then falling through `Array.isArray(d.slots) ? d.slots : []` → always empty. The fallback `_SLOTS_FALLBACK` only kicked in if the request *errored*, not if the response shape was wrong.

**Impact:** No prospect can complete a booking. Conversion blocker.

**Fix (staged in `book.html`):** Rewrote `fetchSlots` to fetch the config once, then compute slot times client-side from `start_hour`, `end_hour`, `slot_duration_min`, filter by `days_available`/`blocked_dates`, and drop slots in the past for today. Falls back to `_SLOTS_FALLBACK` if the endpoint errors entirely.

### 2. MEDIUM — Resources page renders raw HTML entities

**Symptom:** Article descriptions on `/resources` show `today&#8217;s edition` instead of `today's edition`. Same for `&#8220;`/`&#8221;` (smart quotes).

**Root cause:** Some feed items arrive with descriptions already entity-encoded. `escHtml` then escapes the leading `&` to `&amp;`, producing `&amp;#8217;` which renders as the visible `&#8217;`.

**Fix (staged in `resources.html`):** Added `decodeEntities()` (uses textarea trick to decode), wrapped through new `safeText()` which decodes once then re-escapes. Applied to source label, title, summary.

### 3. LOW — `_archive/v3-pre-redesign/*.html` clobbered

The historical pre-redesign snapshots in `_archive/v3-pre-redesign/` were overwritten with current dark-theme versions during the prior recovery work — ~5,000 deletions across 10 files. Doesn't affect production but loses the historical reference.

**Fix:** `git checkout HEAD -- _archive/` once the index lock clears. (Not yet applied — see below.)
